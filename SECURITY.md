# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates. 
Only supported versions will receive patches for newly discovered vulnerabilities.

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

If you are using an unsupported version, please upgrade to the latest supported release to receive security updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

### How to Report
Please report vulnerabilities by emailing the maintainers or opening a **private security advisory** through GitHub.

Include the following information when possible:
- A clear description of the vulnerability
- Steps to reproduce the issue
- The affected version(s)
- Potential impact of the vulnerability
- Any suggested fixes or mitigations (if known)

### Response Time
- We aim to acknowledge vulnerability reports within **48–72 hours**.
- After verification, we will provide updates on the progress of the fix.
- Critical vulnerabilities will be prioritized and addressed as quickly as possible.

### Responsible Disclosure
Please **do not publicly disclose the vulnerability** until we have investigated the issue and released a fix. This helps protect users of the project.

### What to Expect
After a report is submitted:
1. The maintainers will review and validate the vulnerability.
2. If confirmed, a fix will be developed and tested.
3. A security patch will be released and documented.
4. The reporter may be credited for responsible disclosure (if desired).

If the report is determined not to be a vulnerability, we will explain the reasoning and close the report.

Thank you for helping keep this project secure.
